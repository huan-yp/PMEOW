# 设计

## UI

轻逻辑，单纯展示逻辑即可。

支持 Web 和 App

### 通信

Websocket 收

- 资源状态信息推送（更新概览）
- 事件推送（调用通知等）

### 节点状态和历史

#### 显示 UI

- 自适应折线图模式
  - 每个 GPU 的显存占用（按人员归属）
  - 其它信息变化趋势图（注意
  - 所有折线图基本能复用同一套基础模式，前端只请求一次历史，做缓存，然后根据拿到的最新推送只更新最新一个窗口的情况
    - 提供一个按钮刷新历史缓存（主要做调试用）
  - 关于数据点跨度要能够自适应之类的。
- 横条模式（用于每个 GPU 当前的占用/预留情况）
  - 按人员分组涂色

#### 概览

- 实时折线图功能

- 历史回放
  - 概览默认为当前时间点，可以改，改了之后几个折线图都显示历史情况（历史那个时刻的概览情况，详细度可以降低）

#### 历史

- 历史查询
- 节点详情的任务版面只写个大概，主要 tab 放到任务

#### 进程

- 默认展示当前的情况，提供基础的排序功能
- 可以选择一个时间点看那个时候的快照（如果保留了窗口）
- 可以选择一个 PID 去追溯这个进程的生命周期和占用（可以考虑复用折线图）

### 人员

基本假设

- 同一人员在同一机器上只有一个账号

用折线做一下历史占用情况。

同任务调度列一下人员的任务。

### 任务调度

- 折线图展示一下任务的历史占用情况
- 展示一下任务的一些信息。

### 告警管理

分页查一下历史告警的事件，这个被持久化维护了。
