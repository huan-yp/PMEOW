# Web 控制台页面地图

这份文档只回答一件事：当前桌面控制台里，功能入口分别在哪。完整操作流程会下沉到其他主题页，例如节点接入、人员绑定和移动端访问。

## 登录与入口

桌面控制台有两种典型访问方式：

- 开发模式访问 `http://localhost:5129`
- 生产模式默认绑定 `0.0.0.0:17200`；本机访问 `http://localhost:17200`

当前管理员登录模型比较简单：

- 首次登录会初始化管理员密码。
- 后续登录使用同一个管理员密码。
- 登录成功后，浏览器会用 JWT 访问 REST API 和桌面端 Socket.IO namespace。

当前左侧导航包含：

- 控制台
- 节点
- 人员
- 钩子规则
- 告警
- 任务调度
- 安全审计
- 设置

## 控制台

“控制台”页是值班时最先看的页面，主要回答两个问题：

- 当前有多少台服务器在线。
- 当前集群的 GPU 使用分布是什么样。

当前页面包含：

- 顶部摘要，例如服务器总数、在线台数、当前时间。
- 每台服务器的概览卡片。
- 按用户聚合的 GPU 使用分布。

如果当前还没有任何服务器记录，页面会直接提示你先去“节点”页添加服务器。

## 节点

“节点”页负责维护服务器清单。当前支持的操作有：

- 新增服务器。
- 编辑服务器。
- 删除服务器。
- 测试 SSH 连接。
- 上传私钥文件。

如果你计划让某台机器以 Agent 方式接入，建议从一开始就把这条服务器记录的 `host` 设置成该节点的真实 hostname。这样 Agent 连接到服务端时，系统才能自动完成 hostname 绑定。

### 节点详情

点击任意服务器卡片后会进入节点详情页。页面顶部会展示：

- 数据源类型徽标：`SSH` 或 `Agent`
- 当前状态：在线、连接中、异常、离线
- 最后上报时间、hostname、uptime 等节点信息

详情页按标签页展示不同维度的信息。

#### 概览

概览标签会显示：

- CPU 和内存仪表盘
- GPU 使用情况和显存分配条
- 过去 30 分钟的 CPU、内存、网络、磁盘速率图
- 系统信息，例如 hostname、kernel、uptime 和 load average
- 当前节点上的人员活动和已解析 GPU allocation

#### 任务

这个标签只会出现在 Agent 节点上，也就是服务器 `sourceType` 已经切换为 `agent` 的情况下。这里主要是该节点本地队列在服务端的镜像视图。

如果你已经部署了 Agent，但没有看到这个标签，首先怀疑 hostname 绑定是否成功，或当前 live session 是否已经超时断开。

#### 进程

“进程”标签用于查看进程与 GPU 归属审计结果，适合排查：

- 哪些进程占用了 GPU
- 它们属于 PMEOW 任务、普通用户进程还是未知进程
- 是否带有可疑原因

#### Docker

如果节点上运行了 Docker，这里会列出容器信息，方便把节点资源使用情况和容器活动联系起来。

## 人员

“人员”页是 people 相关能力的桌面端入口：

- 人员目录会显示每个人的当前显存、运行中任务数、排队任务数和活跃节点数。
- “添加人员”会进入向导式创建页面。
- 点击某个人员卡片会进入详情页，查看绑定关系、显存时间线、关联任务和移动端令牌状态。

完整的人员创建、绑定迁移和令牌生命周期，请继续阅读 [people-and-access.md](people-and-access.md)。

## 钩子规则

“钩子规则”页用于管理自动动作。当前支持：

- 新建、编辑、启用、停用和删除规则
- 测试动作
- 查看执行日志

如果你已经配置了外部 Webhook 或本地脚本，这是排查钩子没有按预期执行时最先要看的页面。

## 告警

“告警”页展示历史告警记录，当前重点支持：

- 按页浏览告警历史
- 查看指标值与阈值的对比
- 对活跃告警执行抑制

抑制后，记录仍然保留在历史中，但会显示为已忽略到某个时间点。

## 任务调度

“任务调度”页是跨 Agent 节点的操作总览。页面会按服务器分组展示三类任务：

- queued
- running
- recent

当前支持的操作包括：

- 暂停某个节点的任务队列
- 恢复某个节点的任务队列
- 取消 queued 或 running 任务
- 对 queued 任务执行优先级调整

这里展示的是服务端维护的任务镜像，不是任务日志全文。任务 stdout 和 stderr 仍然保存在 Agent 节点本地，需要通过 `pmeow-agent logs` 查看。

## 安全审计

“安全审计”页用于查看和处理安全事件，当前页面提供：

- `serverId` 筛选
- `resolved` 筛选
- `hours` 时间窗口筛选
- 把事件标记为安全

当前已接入的核心检测主要有两类：

- 挖矿或可疑关键词命中
- 无归属 GPU 占用持续超过阈值

“标记安全”只会把安全事件记录标记为已处理，不会自动终止目标进程，也不会修改节点上的真实工作负载。

## 设置

“设置”页目前分成几个模块：

- 数据采集：刷新间隔、历史保留天数
- 告警阈值：CPU、内存、磁盘、挂载点、默认抑制天数
- 安全审计：挖矿关键词、无归属 GPU 持续分钟阈值，以及预留的高 GPU 利用率字段
- Agent 部署说明：页面内的简短提示
- 对外 API：是否启用、端口、token
- Web 密码：用于修改管理员密码

需要特别区分：

- 挖矿关键词和无归属 GPU 持续分钟阈值，当前已经参与检测链路。
- 高 GPU 利用率阈值和持续分钟字段，目前仍然更接近预留设置项，不应理解为已经完整生效的安全规则。
