# Web 控制台使用指南

这份文档按左侧导航顺序介绍当前 Web 控制台中的主要页面，适合管理员、实验室值班同学和需要处理任务/安全事件的维护者。

## 登录与导航

Web 控制台有两种典型访问方式：

- 开发模式访问 `http://localhost:5173`
- 生产模式访问 `http://localhost:17200`

当前登录模型比较简单：

- 首次登录会初始化密码
- 后续登录使用同一个管理员密码
- 登录成功后，浏览器会用 token 访问 REST API 和 UI Socket.IO namespace

左侧导航当前包含：

- 概览
- 服务器
- 钩子规则
- 告警
- Tasks
- Security
- 设置

## 概览

“概览”页是值班时最先看的页面，主要回答两个问题：

- 当前有多少台服务器在线
- 整个集群的 GPU 使用分布是什么样

页面包含两类信息：

- 顶部摘要：服务器总数、在线台数、当前时间
- 列表卡片：每台服务器的 CPU、内存、网络、磁盘和 GPU 概览

如果当前还没有任何服务器记录，页面会直接提示你先去“服务器管理”页添加服务器。

## 服务器

“服务器”页负责维护节点清单。当前支持的操作有：

- 新增服务器
- 编辑服务器
- 删除服务器
- 测试 SSH 连接
- 上传私钥文件

表单里最关键的字段是：

- 名称
- 地址 `host`
- 端口
- 用户名
- SSH 私钥路径

如果你计划让某台机器以 Agent 方式接入，建议从一开始就把这条服务器记录的 `host` 设置成该节点的真实 hostname。这样 Agent 连接到服务端时，系统才能自动完成 hostname 绑定。

## 服务器详情

点击任意服务器卡片后会进入“服务器详情”页。这个页面会按标签页展示不同维度的信息。

### 概览标签

概览标签会显示：

- CPU 和内存仪表盘
- 网络吞吐
- GPU 使用情况
- GPU allocation 条形图
- 过去 30 分钟的 CPU、内存、网络、磁盘 IO 历史曲线
- 系统信息，例如 hostname、kernel、uptime、load average

### Tasks 标签

这个标签只会出现在 Agent 节点上，也就是服务器 `sourceType` 已经切换为 `agent` 的情况下。

当前会展示：

- 排队任务数量
- 运行中任务数量
- 当前 queued/running 任务的 task ID 列表

如果你已经部署了 Agent，但没有看到这个标签，首先怀疑 hostname 绑定是否成功。

### 进程标签

“进程”标签用于查看进程与 GPU 归属审计结果，适合排查：

- 哪些进程占用了 GPU
- 它们属于 PMEOW 任务、普通用户进程还是未知进程
- 是否带有可疑原因

### Docker 标签

如果节点上运行了 Docker，这里会列出容器信息，方便把节点资源使用情况和容器活动联系起来。

## 钩子规则

“钩子规则”页用于管理自动动作。当前页面支持：

- 新建规则
- 编辑规则
- 启用/停用规则
- 删除规则
- 测试动作
- 查看执行日志

日志会以抽屉形式打开，展示最近的执行结果、触发时间和错误信息。

如果你已经配置了外部 Webhook 或本地脚本，这是排查钩子没有按预期执行时最先要看的页面。

## 告警

“告警”页展示历史告警记录，当前重点支持：

- 按页浏览告警历史
- 查看指标值与阈值的对比
- 对活跃告警执行抑制

当前页面直接提供 1、3、7、30 天四种抑制时长快捷按钮。抑制后，记录仍然保留在历史中，但会显示为已忽略到某个时间点。

## Tasks

“Tasks” 页是跨 Agent 节点的操作总览。页面会按服务器分组展示三类任务：

- queued
- running
- recent

当前支持的操作包括：

- 暂停某个节点的任务队列
- 恢复某个节点的任务队列
- 取消 queued 或 running 任务
- 对 queued 任务执行优先级调整

要注意两点：

- 页面展示的是服务端维护的任务镜像，不是任务日志全文。
- 日志仍然保存在 Agent 节点本地，需要通过 `pmeow-agent logs` 查看。

## Security

“Security” 页用于查看和处理安全事件，当前页面提供：

- `serverId` 筛选
- `resolved` 筛选
- `hours` 时间窗口筛选
- 把事件标记为安全

当前已接入的核心检测主要有两类：

- 挖矿或可疑关键词命中
- 无归属 GPU 占用持续超过阈值

“标记安全”只会把事件标记为已处理，不会自动终止目标进程，也不会修改节点上的真实工作负载。

## 设置

“设置”页目前分成几个模块：

- 数据采集：刷新间隔、历史保留天数
- 告警阈值：CPU、内存、磁盘、挂载点、默认抑制天数
- 安全审计：挖矿关键词、无归属 GPU 持续分钟阈值，以及预留的高 GPU 利用率字段
- Agent 部署说明：页面内的简短提示
- 对外 API：是否启用、端口、token
- Web 密码：仅在 Web 模式显示，用于修改登录密码

需要特别区分：

- 挖矿关键词和无归属 GPU 持续分钟阈值，当前已经参与检测链路。
- 高 GPU 利用率阈值和持续分钟字段，目前仍然更接近预留设置项，不应理解为已经完整生效的安全规则。

## 推荐的日常操作流

对值班管理员来说，一个比较稳的日常顺序是：

1. 先看“概览”，确认有没有节点离线或 GPU 使用异常。
2. 再看“Tasks”，确认是否有大量任务堆积或队列被暂停。
3. 接着看“Security”和“告警”，处理需要人工确认的事件。
4. 出现单节点问题时，再进入“服务器详情”查看进程、GPU allocation 和历史曲线。

如果页面行为和你的预期不一致，或者某个操作看起来成功但数据没有变化，请继续阅读 [troubleshooting.md](troubleshooting.md)。
