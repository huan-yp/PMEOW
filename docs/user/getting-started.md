# 用户快速开始

这条路径的目标不是完成所有细节部署，而是在 5 到 10 分钟内确认 PMEOW 的三条主链路都通了：Web 登录、第一台节点、桌面控制台。

## 你会完成什么

- 启动 Web 服务端。
- 完成首次管理员登录。
- 接入第一台 SSH 或 Agent 节点。
- 在“控制台”“节点”“任务调度”里看到当前状态。
- 可选地验证“人员”和移动端入口。

## 先理解当前运行模型

PMEOW 当前采用两层运行模型：

- Web 服务端负责监控汇聚、告警、钩子、Agent 接入、任务镜像和最小控制面。
- Python Agent 运行在计算节点本地，负责指标采集、GPU 归属识别、本地任务队列和自主调度。

这意味着服务端是观察者和干预点，但不负责替节点做二次排队调度。任务提交和本地执行仍然以 Agent 为中心。

## 前置条件

- Node.js 20+
- pnpm 9+ 或 npm 10+
- Python 3.10+，仅在要接入 Agent 节点时需要
- Linux 计算节点；GPU 相关功能依赖 `nvidia-smi`

## 1. 先把 Web 跑起来

最直接的试跑方式是从仓库根目录启动开发环境：

```bash
pnpm install

# 终端 1
pnpm dev:web

# 终端 2
pnpm dev:ui
```

开发模式下：

- Web 后端默认监听 `17200`
- UI 开发服务器默认监听 `5173`
- 浏览器访问 `http://localhost:5173`

如果你更想验证接近生产的启动方式，也可以执行：

```bash
pnpm build:web
pnpm start:web
```

这时直接访问 `http://localhost:17200`。

## 2. 完成第一次管理员登录

第一次打开登录页时，如果数据库里还没有管理员密码，这次输入会被当作初始化密码并写入数据库。

为了先完成试跑，你只需要确认两件事：

- 登录后页面能正常进入控制台。
- 刷新一次页面后不会立刻退回登录页。

如果你后续要长期部署，再去看 [web-server.md](web-server.md) 里的 `JWT_SECRET`、数据库路径和密码重置说明。

## 3. 接入第一台节点

你可以先选一种最容易成功的路径。

### 路径 A：先接一台 SSH 节点

适合先验证监控链路。

1. 打开“节点”页面。
2. 新增服务器记录，填写名称、主机地址、端口、用户名和 SSH 私钥路径。
3. 点击“测试连接”，确认服务端能够通过 SSH 读取目标主机。
4. 保存后等待一次采集周期，再回到“控制台”确认卡片开始刷新。

### 路径 B：直接接一台 Agent 节点

适合你已经准备验证本地任务队列、自主调度和 GPU 归属。

1. 先在 Web 控制台创建对应的服务器记录，`host` 最好直接写成节点真实 hostname。
2. 在目标机器安装 Agent：

```bash
cd agent
python3 -m venv .venv
. .venv/bin/activate
pip install -e ".[dev]"

export PMEOW_SERVER_URL=http://your-server:17200
pmeow-agent run
```

`pmeow-agent daemon` 仍然是兼容别名，但当前更推荐写成 `pmeow-agent run`。

如果服务端在 `servers.host` 中找到了唯一精确匹配的 hostname，节点会自动切换到 Agent 模式，并开始显示任务队列与 GPU allocation 数据。

## 4. 验证桌面控制台

第一次试跑后，至少确认下面几点：

- “控制台”页能看到服务器数量和在线状态。
- “节点”页能看到你刚添加的服务器记录。
- SSH 节点能够看到 CPU、内存、磁盘和网络数据。
- Agent 节点能够在“任务调度”中出现对应服务器分组。
- Agent 节点的“节点详情”页会出现“任务”标签页。

## 5. 可选：顺手验证 people 和移动端

如果你想顺便确认新版人员和访问入口也能走通，可以继续做一个最小验证：

1. 打开“人员”页，新建一条人员记录。
2. 进入该人员详情页，创建一个移动端访问令牌。
3. 如果你使用手机浏览器，可以让个人用户访问 `/m/me`。
4. 如果你使用 Android APK 或 Capacitor 原生壳，首次进入会先落到 `/connect`，再选择个人模式或管理员模式。

## 第一次上线的推荐顺序

为了减少排障复杂度，建议按这个顺序推进：

1. 先单独启动 Web 服务端并完成登录。
2. 再接入一台 SSH 节点，确认基础监控链路正常。
3. 然后接入一台 Agent 节点，确认 hostname 绑定成功。
4. 再补人员目录、移动端令牌、任务队列、安全审计和钩子规则等进阶能力。
