# 移动端入口

PMEOW 当前有两类移动端入口，它们共用同一套 UI，但认证方式不同：管理员移动端沿用管理员登录，个人移动端使用独立的人员令牌。

## 两类移动端入口

### 管理员移动端

路径：`/m/admin`

管理员移动端沿用桌面端同一套管理员认证。它更像是桌面控制台的窄屏值班视图，适合在手机上查看集群摘要、任务和节点状态。

提供的页面：

| 路径 | 说明 |
| --- | --- |
| `/m/admin` | 集群概览（节点、运行任务、排队任务） |
| `/m/admin/tasks` | 跨节点任务列表 |
| `/m/admin/nodes` | 节点在线状态 |
| `/m/admin/notifications` | 管理员通知 |

### 个人移动端

路径：`/m/me`

个人移动端使用独立的 Person Token 认证，不依赖管理员 JWT。它面向实验室成员查看自己的任务、节点和通知。

提供的页面：

| 路径 | 说明 |
| --- | --- |
| `/m/me` | 个人概览（运行任务、排队任务、绑定节点、未读通知） |
| `/m/me/tasks` | 自己的任务列表，可取消自己的任务 |
| `/m/me/nodes` | 绑定的节点列表 |
| `/m/me/notifications` | 个人通知流 |
| `/m/me/settings` | 通知偏好设置 |

当前个人设置页支持开关以下通知类型：

- 任务开始
- 任务完成
- 任务失败
- 任务取消
- 节点状态变更
- GPU 可用

## 浏览器访问与原生连接页

当前仓库同时支持两种访问方式：

- 手机浏览器直接访问移动端路由
- 通过 Capacitor 打包的 Android 原生壳访问

这两种方式最容易混淆的点是：`/connect` 连接页主要服务于原生壳和“尚未保存服务器地址”的场景，不是所有浏览器访问都会先出现它。

### 手机浏览器

- 管理员通常直接打开 `/m/admin`，并复用管理员登录态。
- 个人用户通常直接打开 `/m/me`，由当前部署方式决定是否已经保存过对应会话。

### Android / Capacitor 原生壳

在原生环境中，如果本地还没有保存服务器地址，应用会先进入 `/connect` 连接页。连接页支持两种模式：

- 个人模式：填写服务器地址和人员令牌，应用会调用 `GET /api/mobile/me/bootstrap` 并带上 `X-PMEOW-Person-Token`。
- 管理员模式：填写服务器地址和管理员密码，应用会调用 `POST /api/login` 获取 JWT。

这意味着当前原生移动端的首次连接不是直接从 `/m/me` 或 `/m/admin` 开始，而是先做一次服务器地址和认证材料录入。

## 人员令牌怎么获取和管理

个人移动端令牌由管理员在桌面端人员详情页管理。当前支持：

- 创建令牌
- 轮换令牌
- 吊销令牌
- 查看令牌是否存在以及最近使用时间

令牌创建或轮换时，明文只会显示一次。泄露时应立即轮换或吊销。

更完整的人员、绑定关系和令牌流程，请继续阅读 [people-and-access.md](people-and-access.md)。

## Android APK

PMEOW 提供通过 Capacitor 打包的 Android APK。APK 内嵌 UI 静态资源，只有 API 请求和 Socket 连接需要访问远程 PMEOW 服务端。

如果你只是以终端用户身份安装 APK，通常只需要拿到管理员构建好的安装包并完成首次连接。

如果你是维护者，需要本地构建 Android 调试包，可以使用：

```bash
pnpm build:apk
pnpm --filter @monitor/ui apk:debug
```

## 常见问题

- 连接失败：先确认服务器地址是否是完整的 PMEOW Web 基础 URL，协议和端口都要写对。
- 令牌无效：确认令牌没有被吊销或轮换，必要时由管理员重新生成。
- 管理员密码错误：原生连接页的管理员模式会直接调用管理员登录接口，密码和桌面端一致。
- HTTPS 证书错误：如果服务器使用自签名证书，Android 可能拒绝连接。正式部署更建议使用受信任证书。
