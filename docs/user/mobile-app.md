# 移动端入口

PMEOW 提供两种移动端入口，均为 Web 应用，适合在手机浏览器或 WebView 中访问。

## 管理员移动端

路径：`/m/admin`

管理员使用桌面端同一套 JWT 认证，无需额外令牌。在窄屏视口打开即可看到为移动端优化的布局。

提供的页面：

| 路径 | 说明 |
| --- | --- |
| `/m/admin` | 集群概览（节点、运行/排队任务计数） |
| `/m/admin/tasks` | 跨节点任务列表 |
| `/m/admin/nodes` | 节点在线状态 |
| `/m/admin/notifications` | 管理员通知 |

## 个人移动端

路径：`/m/me`

个人移动端使用独立的 Person Token 认证，不依赖管理员 JWT。

### 获取令牌

1. 管理员在桌面端打开某个人员的详情页。
2. 在"移动端访问令牌"面板中点击 **创建令牌**。
3. 复制显示的 `pmt_...` 令牌，该令牌创建后仅展示一次。
4. 将令牌发送给对应人员。

### 使用令牌

1. 在手机浏览器打开 `/m/me`。
2. 在输入框填入 `pmt_...` 令牌，点击 **验证令牌**。
3. 验证通过后进入个人首页。

提供的页面：

| 路径 | 说明 |
| --- | --- |
| `/m/me` | 个人概览（运行任务、排队任务、绑定节点、未读通知） |
| `/m/me/tasks` | 自己的任务列表（可取消） |
| `/m/me/nodes` | 绑定的节点列表 |
| `/m/me/notifications` | 个人通知流 |
| `/m/me/settings` | 通知偏好设置 |

### 通知设置

个人用户可以在 `/m/me/settings` 页面开关以下通知类型：

- 任务开始
- 任务完成
- 任务失败
- 任务取消
- 节点状态变更
- GPU 可用（默认关闭）

GPU 可用通知仅在启用后生效，需配合阈值参数（最少可用 GPU 数量和最小可用显存）。

### 令牌管理

管理员在桌面端人员详情页可以执行以下操作：

- **创建令牌** — 为没有令牌的人员首次创建
- **轮换令牌** — 作废旧令牌并生成新令牌
- **吊销令牌** — 永久禁用移动端访问

令牌泄露时应立即吊销或轮换。

## 关于原生应用

当前移动端是纯 Web 实现。代码定义了 `MobileNotificationBridge` 接口，未来如需打包为原生移动应用（如 Capacitor 或 React Native WebView），可对接该接口实现原生推送通知。原生应用打包不在本仓库范围内。

## Android APK 安装

PMEOW 提供通过 Capacitor 打包的 Android APK，将移动端 Web UI 打包为原生应用。APK 内嵌 UI 静态资源，仅 API 请求通过网络访问远程服务器。

### 安装 APK

1. 从发布渠道获取 `app-debug.apk`（或由管理员构建后分发）。
2. 在 Android 设备上允许"安装未知来源应用"。
3. 打开 APK 文件完成安装。

### 首次连接

打开 PMEOW 应用后会显示连接配置页面，需要填写以下信息：

| 字段 | 说明 | 示例 |
| --- | --- | --- |
| **服务器地址** | PMEOW Web 服务器完整 URL（含协议和端口） | `https://lab.example.com:17200` |
| **令牌** | 管理员为你生成的个人访问令牌 | `pmt_a1b2c3d4...` |

填写完成后点击 **连接**，应用会自动验证令牌有效性。验证通过后进入个人首页。

如果需要以管理员身份访问，可以在连接页面切换到 **管理员模式**，使用管理员密码登录。

### 获取令牌

个人访问令牌由管理员在桌面端创建，步骤同上方"获取令牌"章节。

### 切换服务器

在个人移动端页面中，如果需要连接到不同的服务器，清除应用数据或重新安装即可回到连接配置页面。

### 常见问题

- **连接失败** — 检查服务器地址是否正确、网络是否可达、端口是否开放。
- **令牌无效** — 确认令牌未被吊销或轮换，联系管理员重新生成。
- **HTTPS 证书错误** — 如果服务器使用自签名证书，Android 可能拒绝连接。建议使用受信任的证书，或参考开发文档配置 Network Security Config。
