# 移动端 App

PMEOW 当前的移动端是独立的 React Native Android 应用，不再依赖浏览器移动路由或 Capacitor 外壳。

App 内保留两种登录模式：

- 管理员模式：输入后端 URL 和管理员密码
- 个人模式：输入后端 URL 和人员访问令牌

首次打开时，应用会先进入连接页。连接配置、登录态和通知偏好都只保存在当前设备本地。

## 页面结构

### 管理员模式

管理员模式适合值班和巡检，当前提供三块主页面：

| 页面 | 说明 |
| --- | --- |
| 看板 | 查看节点在线状态、资源摘要、运行任务和排队任务 |
| 告警 | 查看告警、安全事件和最近任务事件 |
| 设置 | 管理本地通知开关、查看通知收件箱、退出登录 |

### 个人模式

个人模式面向实验室成员查看自己的工作负载，当前提供三块主页面：

| 页面 | 说明 |
| --- | --- |
| 首页 | 查看自己可访问节点的状态、资源摘要和机器卡片 |
| 我的任务 | 查看自己的任务列表，并取消仍可取消的任务 |
| 设置 | 管理任务通知和空闲机器订阅，查看通知收件箱，退出登录 |

首页中的机器卡片可以继续进入机器详情页，查看该节点的最新摘要、运行任务和排队任务。

## 通知模型

移动端通知是本地系统通知，不依赖远程推送服务。当前支持：

- 管理员：告警通知、安全事件通知、任务事件通知
- 个人：任务事件通知
- 个人：空闲机器订阅提醒
- 本地通知收件箱，保留最近通知历史

需要注意的边界：

- 通知偏好和空闲订阅只保存在本机
- 仅保证 App 在前台或后台仍存活时可以触发本地通知
- 如果 App 被系统彻底杀死，不保证还能继续收到通知

## 人员令牌怎么获取和管理

个人移动端令牌由管理员在桌面端人员详情页管理。当前支持：

- 创建令牌
- 轮换令牌
- 吊销令牌
- 查看令牌是否存在以及最近使用时间

令牌创建或轮换时，明文只会显示一次。泄露时应立即轮换或吊销。

更完整的人员、绑定关系和令牌流程，请继续阅读 [people-and-access.md](people-and-access.md)。

## Android APK

PMEOW Mobile 通过 React Native Android 工程构建 APK。应用本身运行在设备上，只有 API 请求和 Socket 连接需要访问远程 PMEOW 服务端。

如果你只是以终端用户身份安装 APK，通常只需要拿到管理员构建好的安装包并完成首次连接。

如果你是维护者，需要本地构建 Android 安装包，可以使用：

```bash
npm run build:apk
```

详细前置依赖、产物路径和安装方式见 [developer/build/构建 APK.md](../developer/build/%E6%9E%84%E5%BB%BA%20APK.md)。

## 常见问题

- 连接失败：先确认服务器地址是否是完整的 PMEOW Web 基础 URL，协议和端口都要写对。
- 令牌无效：确认令牌没有被吊销或轮换，必要时由管理员重新生成。
- 管理员密码错误：移动端管理员模式直接调用管理员登录接口，密码和桌面端一致。
- 没有收到通知：先确认系统通知权限已授予，并确保 App 仍在前台或后台存活。
- HTTPS 证书错误：如果服务器使用自签名证书，Android 可能拒绝连接。正式部署更建议使用受信任证书。
